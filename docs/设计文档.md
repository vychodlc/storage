这个管理系统是“古楼山竹筷厂”（以下称“该厂”）的生产管理系统。该厂会将竹丝加工为不同尺寸的竹筷。

工厂的日常工作项目主要分为以下部分
1. 进货：向各个竹丝供应商进货
   1. 供应商有多个，需要管理供应商的联系方式、负责人、地点
   2. 司机有多个，如果是物流的话特殊标记即可
   3. 每次进货可能会进多个尺寸的竹丝：比如 6mm * 200mm * 15捆 + 5mm * 200mm * 15捆
   4. 需要记录汇款的信息，有可能是一次性全付/多次付（如果是多次付的话，每次都算一次流水，但是关联的是同一笔订单）
   5. 需要支持新建每次进货单
2. 生产：
   1. 每日消耗了多少竹丝（消耗以捆为单位），需要有操作人
   2. 每日生产了多少袋竹筷，需要有操作人
3. 出货：
   1. 客户有多个，需要管理客户的联系方式、负责人、地点
   2. 要管理司机，配置司机的联系方式、姓名、车牌号
   3. 需要记录订单进度（已发货、未汇款完全、汇款全部完成），汇款的信息，有可能是一次性全付/多次付（如果是多次付的话，每次都算一次流水，但是关联的是同一笔订单）
   4. 需要支持新建每次出货单
4. 工人管理
   1. 司机工：负责进货。注意：出货的司机是第三方平台的，每次都不一样，暂不考虑管理
   2. 普通工人：负责日常生产，需要记录每日产量，每日考勤（半天/全天）
   3. 临时工：主要负责卸货、上货
      1. 工资是日结的，所以不需要进行出勤管理
5. 出勤管理
   1. 按照月维度，是一个表格，每一行是一个工人的一个月的出勤情况，假如是30天的话，每个单元格默认是“未上工”，点击一次变成“全天”，再点击一次变成“半天”，再点击一次变成“未上工”
   2. 每个工人名字旁边有一个按钮，点击会弹出该员工的当月出勤天数，并且下面有个输入框是日工资，输入完之后在下方计算出该员工的当日工资
6. 流水表
   1. 进货订单
   2. 出货订单
   3. 工人工资
   4. 出货的司机费用
